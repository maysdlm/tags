As tags são usadas para informar ao navegador a estrutura do site. Ou seja: quando se escreve um código em HTML, as tags serão interpretadas pelo navegador, produzindo assim a estrutura e o conteúdo visual da página.

Abaixo são apresentadas algumas tags:

header

O elemento < header > representa conteúdo introdutório para seu conteúdo de seção ancestral mais próximo ou elemento raiz de seção. O header normalmente contém um grupo de auxiliares introdutórios ou de navegação. Quando o conteúdo de seção ancestral mais próximo ou elemento raiz de seção é o elemento body, então ele se aplica a toda a página.

Exemplo de uso de < header >:



        < header >
            < h1 >Título da página< /h1 >
            < h2 >Subtítulo da página< /h2 >
        < /header >

Nota: Diferentemente da tag < head >, é possível declarar mais de um < header > por página.




section

O elemento < section > representa uma seção genérica de um documento ou aplicativo. Uma seção, neste contexto, é um agrupamento temático de conteúdo. Cada seção deve ser identificada, tipicamente incluindo um título (elemento h1-h6) como uma criança do elemento section.

Exemplo de uso de < section >:



        < section >
            < h3 >Seção 1< /h1 >
            < p >Texto< /p >
        < /section >



article

O elemento < article > representa uma composição completa ou independente em um documento, página, aplicativo ou site e que é, em princípio, distribuível ou reutilizável independentemente, por exemplo, em sindicância. Este pode ser um post no fórum, uma revista ou artigo de jornal, uma entrada no blog, um comentário enviado pelo usuário, um widget interativo ou gadget, ou qualquer outro item independente de conteúdo. Cada article deve ser identificado, tipicamente incluindo um título (elemento h1-h6) como uma criança do elemento article.

Exemplo de uso de < article >:



        < article >
            < h3 >Título do artigo 1< /h3 >
            < p >Texto< /p >
        < /article >



nav

O elemento < nav > representa uma seção de uma página que se conecta a outras páginas ou a partes dentro da página: uma seção com links de navegação.

Exemplo de uso de < nav >:


        < nav >
            < ul >
            < li >< a href=”#”>pagina 1< /a >< /li >
            < li >< a href=”#”>pagina 2< /a >< /li >
            < /ul >
        < /nav >

Nota: Podemos declarar o < nav > em qualquer lugar do documento que contenha uma lista de links, inclusive no < header >.




aside

O elemento < aside > representa uma seção de uma página que consiste em conteúdo tangencialmente relacionado ao conteúdo em torno do elemento de lado, e que poderia ser considerado separado desse conteúdo. Tais seções são frequentemente representadas como barras laterais na tipografia impressa.

Exemplo de uso de < aside >:


        < aside >
            < nav >
                < ul >
                    < li >Link< /li >
                < /ul >
            < /nav >
        < /aside >



main

O elemento < main > representa o conteúdo principal do corpo de um documento ou aplicativo. A área de conteúdo principal consiste em conteúdo diretamente relacionado ou se expande sobre o tópico central de um documento ou funcionalidade central de um aplicativo.

Exemplo de uso de < main >:


        < main >
            < h2 >Titulo< /h2 >    
                < p >Texto< /p >
                    < article >
                           < h3 >Subtítulo< /h3 >
                    < /article >
        < /main >



figure

O elemento < figure > representa algum conteúdo de fluxo, opcionalmente com uma legenda, que é auto-contida (como uma frase completa) e é tipicamente referenciada como uma única unidade a partir do fluxo principal do documento.

Exemplo de uso de < figure >:



        < figure >
                < img src=”http://meusite.com.br/assets/imagem.jpg” alt=”Imagem”>
        < /figure >



footer

O elemento < footer > representa um rodapé para seu conteúdo de secção ancestral mais próximo ou elemento raiz de secção. Um rodapé normalmente contém informações sobre sua seção, como quem escreveu, links para documentos relacionados, dados de direitos autorais e afins.

Exemplo de uso de < footer >:



        < footer >
            < p >Escrito por ...< /p >
            < p >Publicado em 16/06/2020< /p >
        < /footer >



a

Se o elemento < a > tem um atributo href, então ele representa um hiperlink (uma âncora de hipertexto) rotulado pelo seu conteúdo.

Exemplo de uso de < a >:





        < a href=”http://www.site.com.br” alt=”Site”>Site< /a >



em

O elemento < em > representa ênfase de estresse de seu conteúdo.

Exemplo de uso de < em >:





        < p >Você < em >tem certeza< /em > que essa definição está correta?< /p >



strong

O elemento < strong > representa forte importância, seriedade ou urgência para seu conteúdo.

Exemplo de uso de < strong >:




        < p >Compreender esses elementos HTML é importante porque
        < strong >possibilita o desenvolvimento de soluções web modernas< /strong >.< /p >



cite

O elemento < cite > representa uma referência a um trabalho criativo. Ele deve incluir o título da obra ou o nome do autor (pessoa, pessoa ou organização) ou uma referência de URL, ou uma referência em forma abreviada de acordo com as convenções usadas para a adição de metadados de citação.

Exemplo de uso de < cite >:



        < p >
            < cite >http://br.site.com/< /cite >.
        < /p >



figcaption

O elemento < figcaption > representa uma legenda ou legenda para o resto do conteúdo do elemento figura pai do elemento de figcapção, se houver.

Exemplo de uso de < figcaption >:



        < figure >
            < img src=”http://meusite.com.br/assets/imagem.jpg” alt=”Imagem”>
                < figcaption >Figura 1. Imagem< /figcaption >
        < /figure >
